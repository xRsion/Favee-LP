# Favee スケジュール管理システム

イベント内容の追加・削除・編集を簡単に管理できるシステムです。

## 📁 ファイル構成

```
├── schedule.html              # メインのスケジュールページ
├── admin.html                # 管理者用編集インターフェース
├── assets/
│   ├── data/
│   │   └── events.json       # イベントデータファイル
│   └── js/
│       └── schedule.js       # スケジュール管理JavaScript
└── README_schedule_management.md
```

## 🚀 使い方

### 1. 通常の表示（ユーザー向け）
- `schedule.html` を開くとイベントスケジュールが表示されます
- フィルター機能でカテゴリ別に表示可能
- レスポンシブデザインでモバイル対応

### 2. 管理者による編集
- `admin.html` を開いて管理インターフェースにアクセス
- 新しいイベントの追加、既存イベントの編集・削除が可能
- データのエクスポート・インポート機能

## 📝 イベント管理方法

### 新しいイベントを追加する場合

1. `admin.html` を開く
2. 左側のフォームに必要な情報を入力：
   - **タイトル**: イベント名
   - **カテゴリ**: アップデート/イベント/コラボ/メンテナンス
   - **ステータス**: 予定/開催中/終了
   - **日付表示**: ユーザーに表示される日付文字列
   - **ソート用日付**: 並び順用の日付（YYYY-MM-DD形式）
   - **アイコン**: 絵文字（📱🎉🤝🔧など）
   - **詳細内容**: 改行区切りで複数項目を入力
   - **注意書き**: 特記事項があれば入力

3. 「イベントを追加」ボタンをクリック

### 既存イベントを編集する場合

1. `admin.html` の右側のイベント一覧から「編集」をクリック
2. フォームに既存の値が入力されるので、必要な箇所を修正
3. 「イベントを更新」ボタンをクリック

### イベントを削除する場合

1. `admin.html` の右側のイベント一覧から「削除」をクリック
2. 確認ダイアログで「OK」をクリック

## 💾 データの保存

### 自動保存について
- 管理画面での変更は一時的にブラウザ内で保存されます
- **重要**: 永続的な保存には手動でJSONファイルを更新する必要があります

### 手動でのデータ保存手順

1. 管理画面で「エクスポート」ボタンをクリック
2. ダウンロードされた `events.json` ファイルを確認
3. `assets/data/events.json` を新しいファイルで置き換え

### バックアップとリストア

**バックアップ**:
- 管理画面の「エクスポート」ボタンでJSONファイルをダウンロード

**リストア**:
- 管理画面の「インポート」ボタンをクリック
- JSONデータを貼り付けて「インポート」実行

## 🎨 カスタマイズ

### 新しいカテゴリを追加する場合

1. `assets/js/schedule.js` の以下の箇所を編集：
   - `categoryClasses` オブジェクト
   - `categoryLabels` オブジェクト
   - `categoryColors` オブジェクト

2. `schedule.html` のフィルターボタンに新しいカテゴリを追加

3. `admin.html` のセレクトボックスに新しいオプションを追加

### スタイルのカスタマイズ

- `schedule.html` の `<style>` セクションでCSS変更
- Tailwind CSSクラスで色やレイアウト調整

## 🔧 開発者向け機能

### ブラウザコンソールでの操作

```javascript
// 新しいイベントを追加
addEvent({
    title: "新しいイベント",
    category: "event",
    status: "upcoming",
    date: "2025年8月1日",
    dateSort: "2025-08-01",
    icon: "🎉",
    details: ["詳細1", "詳細2"]
});

// イベントを更新
updateEvent(1, { title: "更新されたタイトル" });

// イベントを削除
deleteEvent(1);

// データをエクスポート
console.log(exportEvents());

// データをインポート
importEvents('{"events": [...]}');
```

## 📱 レスポンシブ対応

- デスクトップ、タブレット、スマートフォンに対応
- モバイルファーストデザイン
- タッチ操作に最適化

## 🔒 セキュリティ注意事項

- `admin.html` は管理者のみがアクセスできる場所に配置
- 本番環境では適切なアクセス制限を設定
- JSONファイルの直接編集時は構文エラーに注意

## 🆘 トラブルシューティング

### イベントが表示されない場合
1. ブラウザの開発者ツールでエラーを確認
2. `assets/data/events.json` の構文が正しいか確認
3. ファイルパスが正しいか確認

### 管理画面が動作しない場合
1. `assets/js/schedule.js` が正しく読み込まれているか確認
2. ブラウザのJavaScriptが有効になっているか確認

### データが保存されない場合
- 管理画面での変更は一時的です
- 「エクスポート」→手動でファイル更新が必要
